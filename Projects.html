Important information

The first three projects shown in this portfolio were developed using the open-source engine Godot, while the next two were developed using Unity, both engines I learned to use self-taught.

Roguelite

The main objective of this game was to experiment with procedural map generation, as well as a dynamic combat system that would vary throughout a run (modifying stats, attacks, etc.). The idea was to combine these elements with platforming mechanics.

One of the biggest challenges in this project was the design and implementation of the random generation algorithm. This algorithm builds a random graph of rooms, taking into account the entrances and connections available to each room. Once the map is generated, a room containing a random item that enhances the player's character is created.

In the video, two different runs can be seen: the first one appears at the beginning, and the second starts at minute 1:55.

 

Farm game

The goal of this project was to develop the core of a farming simulation game, incorporating essential mechanics such as crafting, planting crops, managing time (days), gathering materials, and more.

One of the biggest challenges was programming the interface and inventory system. The inventory is similar to that seen in games like Minecraft, allowing players to rearrange the order of items and stack up to 99 units of the same item, among other features. Additionally, all in-game objects are destructible, and certain items can be placed in the environment, taking into account both the location and whether or not the object can be placed there.

PCEOMON

This project was developed in collaboration with two classmates, José Francisco Mora Caselles and Eduardo Belmonte Guillamón.

One of the main challenges I faced was joining the development a few weeks after it had already started, which meant working with code that I hadn't written and needed to fully understand. During the first few days, I focused on reading the code, making small implementations, and fixing bugs to ensure I had a complete grasp of its structure and functionality.

The game is based on a turn-based combat system, where a team of up to five members faces off against a boss. Another notable aspect of this project was the development of the game's interface and menus.

Bread or Love

This game was developed during a three-day GameJam in collaboration with two teammates. The main idea was to create a platformer where, instead of directly controlling the character, the player controls the currently equipped weapon.

The biggest challenge was the limited time and the implementation of four different gameplay mechanics in just three days. All the weapons can be seen in the video:

  • First weapon: Shotgun (Min 0:45)
  • Second weapon: Grappling Hook (Min 1:55)
  • Third weapon: Vacuum (Min 2:40)
  • Fourth weapon: Portal Gun (Min 4:30)

Action game

This is an action game inspired by the gameplay of the Zelda series. The main goal was to develop a simple combat system, incorporating a couple of enemies to test their artificial intelligence.

The biggest challenge was manually programming the A* algorithm used for the Slime enemy to efficiently track the player. The objective was to untderstand exactly how it worked.
